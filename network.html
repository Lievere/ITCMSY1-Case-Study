01 Network Topology

Hierarchical Star Topology

A network topology is the logical map of how devices connect and communicate. For a multi-floor office building housing three departments with different access requirements, the right topology is not flat it is layered. IT COMSEE Corp.'s network follows a Three-Tier Hierarchical Star model, the same approach used in enterprise campus networks worldwide.

🌐

Why Hierarchical Star?

The hierarchical model addresses these limitations by segmenting the network into functional layers, enabling improved traffic control, fault isolation, and scalability. Each floor operates as an independent access segment, while centralized routing ensures efficient communication across the organization (Tanenbaum, 2010).

🔵
Core Layer
Server Room
The single Layer 3 switch in the server room. All access switches, servers, the NAS, and the firewall connect here. Handles inter-VLAN routing and 10 GbE uplinks to the server rack. This is the highest-bandwidth point in the network.
🟢
Access Layer
One Switch Per Floor (×3)
48-port switches on each floor connect employee workstations at 1 GbE. They tag traffic with the correct VLAN based on which port a device is plugged into, then forward everything upward to the core. Employees on the same floor but different departments are still logically separated.
🔒
Security Perimeter
Firewall
The FortiGate firewall sits between the internet (ISP uplink) and the core switch. It inspects all inbound and outbound traffic, blocks unauthorized connections, provides VPN access for remote workers, and acts as the organization's primary defense against external threats.
📡
Wireless Layer
One AP Per Floor (Minimum)
802.11ac Wave 2 access points on each floor provide wireless access. Each SSID is mapped to a specific VLAN a device connecting to the "ITCOMSEE-Admin" Wi-Fi network is automatically placed in VLAN 10, with exactly the same access restrictions as a wired Administration workstation.

04 Transmission Media

How the Signal Travels

Transmission media refers to the physical medium through which data signals travel between devices. Each network connection relies on a specific medium, such as copper cabling, fiber optics, or wireless transmission, selected based on bandwidth requirements, distance, and environmental conditions. The IT COMSEE Corp. network design incorporates three types of media, each aligned with a specific operational requirement.

🔌
Cat6A UTP
1 GbE · up to 100m
Horizontal runs from floor switches to every workstation wall port. Supports up to 10 GbE but deployed at 1 GbE for cost efficiency at the edge.
💡
OM4 Multimode Fiber
10 GbE · up to 400m
Vertical backbone runs between floors and into the server room where 10 GbE speeds are needed. Immune to electrical interference ideal near server UPS equipment.
⚡
SFP+ DAC Twinax
10 GbE · up to 5m
Direct-attach copper cables inside the server room connecting each server and the NAS directly to the core switch SFP+ ports. Cheaper than fiber transceivers for short distances.
📡
802.11ac Wave 2
Up to 1.3 Gbps · 5 GHz
Wireless medium for employee laptops, mobile devices, and visitors. Dual-band (2.4/5 GHz). Each SSID is mapped to a VLAN wireless users get the same security policy as wired users in the same department.

Why Not All Fiber, or All Copper?

Fiber optic cabling offers higher bandwidth capacity and is not susceptible to electromagnetic interference. However, it involves higher material costs and requires specialized installation and termination. Deploying fiber to every workstation in an office environment of this scale would significantly increase costs without providing proportional performance benefits, as Gigabit Ethernet over copper remains sufficient for typical user workloads.

Fiber and DAC twinax connections are therefore reserved for high-throughput links within the server environment, where continuous data transfer occurs between servers and storage systems. This selective deployment of transmission media reflects standard enterprise network design practices (Cisco Systems, 2023).

Fiber and DAC twinax are reserved for the links where it actually matters: the server room, where large amounts of data move continuously between servers and storage. This hybrid media approach is standard practice in enterprise campus design. (Cisco Systems, 2023)

Relevance: IPv4 addressing each VLAN gets its own /24 subnet (192.168.10.0, 192.168.20.0, 192.168.30.0, 10.0.99.0). The Layer 3 core switch routes traffic between subnets for authorized inter-VLAN communication (e.g., Operations accessing the File Server VM in the server rack). The FortiGate firewall operates at this layer for NAT, routing, and stateful inspection.
2
Data Link
Relevance: Ethernet frames, MAC address table lookups at each switch, and IEEE 802.1Q VLAN tagging. When an employee on Floor 2 sends traffic, their access switch tags each frame with the correct VLAN ID before forwarding it up the trunk link to the core switch. Spanning Tree Protocol (STP) prevents bridging loops on the dual uplinks from access switches. This is the most operationally significant layer for this design.
1

Relevance: Cat6A copper cabling (electrical signal, 1 GbE to workstations), OM4 multimode fiber (light signal, 10 GbE backbone), SFP+ DAC twinax (10 GbE server room), RJ-45 and LC fiber connectors, patch panels in each floor's comms cabinet, switch ports, and NIC hardware in servers and workstations. A cable fault or bad RJ-45 crimp at this layer stops everything above it from working.
06 TCP/IP & Addressing

TCP/IP & IPv4 Addressing Strategy

IT COMSEE Corp.'s addressing scheme follows IPv4 as the primary protocol, with VLAN-based subnet separation ensuring that Administration, Operations, and IT Support traffic is logically isolated even when sharing the same physical infrastructure.

DHCP Automatic Address Assignment

The Active Directory / LDAP Virtual Machine on Server 1 acts as the DHCP server for all VLANs. The core switch forwards DHCP requests from each VLAN (DHCP relay) to the single centralized DHCP server, which responds with the correct IP range for the requesting VLAN.

This means an employee's laptop is automatically assigned an IP address in the correct subnet the moment it connects whether wired on any floor or wirelessly via an AP. Centralizing DHCP on the AD VM also ensures that hostname registrations (DNS records) are created automatically at assignment time, keeping the internal DNS zone up to date.

DNS Internal Name Resolution

An internal DNS zone itcomsee.local resolves all internal service names to their IP addresses. For example:

  • files.itcomsee.local → 10.0.10.11 (File Server VM)
  • db.itcomsee.local → 10.0.10.12 (Database VM)
  • backup.itcomsee.local → 10.0.10.13 (PBS VM)
  • monitoring.itcomsee.local → 10.0.10.20 (Grafana container)

External DNS queries (e.g., google.com) are forwarded through the FortiGate firewall to the ISP's resolvers or Cloudflare (1.1.1.1). Internal records never leave the corporate network.

IPv6 Future-Proofing Consideration

IPv4 has a theoretical maximum of approximately 4.3 billion unique addresses globally a pool that has been exhausted at the regional level since 2011. While a private organization using RFC 1918 private addressing (192.168.x.x, 10.x.x.x) is insulated from this globally for now, IPv6 adoption is accelerating and will eventually be unavoidable.

For this deployment, IPv4 is the primary protocol. IPv6 is not deployed in Phase 1. However, the recommended approach for Phase 2 (within 12–24 months) is a dual-stack configuration:

  • ▸Assign IPv6 Unique Local Addresses (ULA, prefix fd00::/8) to each VLAN alongside IPv4
  • ▸Enable IPv6 on the FortiGate if the ISP provides a public IPv6 prefix
  • ▸All internal services (DNS, file sharing, AD) should be verified for IPv6 compatibility before enabling

Dual-stack ensures that both old IPv4-only devices and newer IPv6-capable devices can coexist without disruption during the transition period.

Inter-VLAN Routing How Departments Talk to Shared Services

VLANs are by design isolated from each other. An Administration workstation on 192.168.10.x cannot reach an Operations workstation on 192.168.20.x and that is intentional. However, both departments need to reach shared services in the server rack (file server, database, AD authentication).

The Layer 3 core switch handles inter-VLAN routing: it has a virtual interface (SVI Switched Virtual Interface) in each VLAN acting as the gateway. When a workstation needs to reach a server in the server rack subnet, the packet goes to its VLAN gateway on the core switch, which routes it to the correct destination. Routing decisions are enforced by ACLs (Access Control Lists) that permit only the specific communication patterns the organization requires:

  • ✔All VLANs → Server rack (TCP 445 SMB, TCP 389 LDAP, TCP 443 HTTPS) permitted
  • ✔VLAN 30 (IT Support) → VLAN 99 (Management) permitted
  • ✗VLAN 10 → VLAN 20 or VLAN 30 (direct workstation-to-workstation) blocked
  • ✗Any VLAN → VLAN 99 Management (except VLAN 30 IT Support) blocked
07 Input / Output Devices

I/O Devices in the Environment

Input and output devices are the hardware through which users and systems interact with the computing environment. In a networked organization like IT COMSEE Corp., most I/O devices communicate over the network rather than being directly connected to a single computer which means their behavior is governed by both local drivers and network protocols.

⌨ Input Devices
⌨
Keyboard & Mouse
HID (Human Interface Device) class USB peripherals on all employee workstations. Generate hardware interrupts on keypress/movement events that the OS processes via the HID driver stack.
🌐
Network Interface Cards (NIC)
1 GbE NIC on employee workstations; dual 10 GbE NICs (bonded) on each server. The NIC generates a hardware interrupt when an Ethernet frame arrives, triggering the OS network stack. On servers, DMA transfers the received frame directly to RAM without CPU involvement.
📷
IP Security Cameras
PoE-powered IP cameras in common areas stream H.264 video over the Operations VLAN to an NVR (Network Video Recorder) application running as a container on Server 2.
📠
Barcode / RFID Scanners
Used by Operations department for asset tracking. USB or wireless (Bluetooth/Wi-Fi) scanners feed data into an inventory management web application hosted as a Docker container.
🖥
KVM-over-IP Switch
A hardware KVM (Keyboard/Video/Mouse) device in the server rack allows IT Support to access server consoles remotely even when Proxmox is unresponsive. Accessible only from VLAN 99.
🖨 Output Devices
🖨
Network Printers
Department-shared printers connected via TCP/IP (port 9100 raw or 631 IPP). Assigned static IPs within each department's VLAN. All print jobs route through the network no local USB printing. Managed via Windows Print Server role on the File Server VM.
🖥
Employee Monitors
DisplayPort or HDMI output from employee workstations. Not network-connected purely local output. Relevant to I/O because the display controller uses DMA to write framebuffer data to the GPU without CPU involvement for each frame.
🔋
UPS SNMP Output
The APC Smart-UPS sends SNMP traps to the Prometheus monitoring container when battery power is engaged, power quality degrades, or battery capacity falls below threshold enabling automatic VM shutdown sequencing before power is exhausted.
📺
Server Status / iDRAC Console
Dell iDRAC (Integrated Dell Remote Access Controller) on each server provides out-of-band video output and management even when the OS is unresponsive. Accessible from VLAN 99 Management only.
📞
VoIP Handsets
SIP-based desk phones connected over the Operations VLAN. PoE-powered directly from the floor access switch. Voice traffic is prioritized using QoS (Quality of Service) DSCP markings at the switch level to ensure call quality under heavy network load.
08 I/O Control Mechanisms

How the System Controls I/O Traffic

Every time a device reads or writes data from a key press to a 10 GbE packet arriving at a server NIC the operating system must decide how to handle it. Three fundamental mechanisms govern this: polling, interrupts, and Direct Memory Access (DMA). Each has trade-offs in CPU utilization and latency. (Tanenbaum & Bos, 2015)

MECH-01 Hardware Interrupts The Primary Mechanism
▾
What it is: A hardware interrupt is a signal sent by a device to the CPU saying "something happened come deal with it." The CPU stops whatever it was doing, saves its state, and executes an Interrupt Service Routine (ISR) a small piece of code that handles the event.

Where it's used in this environment: Network cards generate an interrupt for every batch of packets received. Keyboard key presses generate interrupts. Disk controllers generate interrupts when a read or write operation completes. The UPS SNMP agent generates a software interrupt (signal) when power status changes.

Why it's efficient: The CPU does not need to constantly check ("poll") whether a device has data. It runs other work and only responds when the device actually needs attention. This is why modern servers can handle hundreds of thousands of network events per second while still running VMs at full speed the CPU is not wasting cycles waiting.

In virtualized environments: Proxmox passes hardware interrupts from physical NICs to the specific VM or container that owns that network interface using IOMMU (Input-Output Memory Management Unit) on the Intel CPU. This allows VMs to handle their own network interrupts without routing through the hypervisor for every packet, reducing latency significantly.
MECH-02 DMA Direct Memory Access (Critical for Server I/O)
▾
What it is: DMA allows a device controller (NIC, NVMe SSD controller, GPU) to transfer data directly into the computer's RAM without involving the CPU for each byte of data. The CPU initiates the transfer, then steps aside while the DMA controller moves the data autonomously. When the transfer is complete, an interrupt notifies the CPU.

Why it matters for this deployment: At 10 GbE speeds, the server NIC can receive 1.25 GB of data every second. Without DMA, the CPU would have to copy every single byte from the NIC to RAM by itself an impossible workload while simultaneously running VMs. DMA makes 10 GbE (and NVMe SSD reads at 3.5 GB/s) practical on a server that is simultaneously running 6–8 virtual machines.

NVMe specifically: The NVMe protocol was designed from the ground up around DMA and parallel queues. Each NVMe SSD can process up to 65,535 simultaneous I/O queues compared to SATA's single queue because DMA lets the SSD controller write data to RAM without serializing through the CPU. This is why VM snapshot operations on ZFS NVMe pools complete in seconds rather than minutes.
MECH-03 Polling When and Why It Is Used
▾
What it is: Polling is the technique where the CPU periodically checks a device's status register to see if it needs attention. Unlike interrupts, polling keeps the CPU actively involved it spends clock cycles repeatedly asking "are you done yet?"

When polling is used in this environment: The Prometheus monitoring container uses polling it queries the Proxmox API, SNMP endpoints, and all VM metrics at a configurable interval (default: 15 seconds). This is appropriate here because monitoring data is not time-critical a 15-second delay in seeing a metric change is acceptable, and interrupt-driven monitoring would generate enormous overhead on every server simultaneously.

High-performance networking (DPDK): There is a specific scenario where polling is used for very high-throughput network applications called kernel-bypass networking (DPDK). In this model, the NIC driver runs entirely in userspace and polls the NIC at full CPU speed, eliminating interrupt latency. This is used in telecommunications and high-frequency trading environments. IT COMSEE Corp. does not need this for 80–120 users and a 10 GbE environment, standard interrupt-driven networking is sufficient and much more CPU-efficient.
MECH-04 QoS Traffic Priority Control
▾
What it is: Quality of Service (QoS) is a network-level I/O control mechanism that assigns priority levels to different types of traffic. When a switch or router's output buffer fills up (congestion), lower-priority packets are dropped or delayed while higher-priority packets are forwarded first.

Where it's applied: The Cisco access switches and core switch are configured with DSCP (Differentiated Services Code Point) markings. VoIP traffic (Operations VLAN) is marked as high priority ensuring call quality is maintained even during a large file upload from another workstation on the same floor. Video surveillance streams are marked as medium priority. General file transfers and web browsing receive best-effort (lowest) priority.

Why this matters: Without QoS, a single employee uploading a 10 GB file to the shared drive could saturate the floor's 1 GbE uplink to the core switch, making VoIP calls drop for everyone else on that floor. QoS ensures the network behaves predictably under load which directly supports the organization's goal of reliable network-based services.
